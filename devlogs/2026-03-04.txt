Title: Movement overhaul - stamina, crouch toggle, lower jump
TL;DR: Reworked player movement: lower jump height, C toggles crouch, and stamina drains when sprinting with a subtle exhaustion vignette.
---

- Added stamina system: drains at 20/s while sprinting, regens at 15/s after 1s delay
- No stamina bar — instead, a dark vignette creeps in from screen edges when stamina drops below 50% (follows existing overlay pattern from proximity.css)
- Stamina hitting 0 forces sprint off until it regenerates

Title: Needs system rework - thirst, water containers, consumables

- Device tama tab now shows HUN/THR/HAP bars + H2O (water container level) + GLS (glass health)

Title: Glass behavior overhaul - persistent damage, repairs, random banging
TL;DR: Glass damage persists after tamas calm down. Players can click-to-repair at containment rooms. Tamas randomly bang on glass even when calm.
---
- Glass damage is now persistent — calming a tama no longer auto-repairs the glass. Only manual repair or night restart fixes it.
- Added click-to-repair interaction: walk up to containment glass, [E] Repair Glass (+10 HP per press). Only shows when glass is damaged and tama isn't escaped.
- Added `repairPartial()` and `takeDamage()` methods to Containment class for clean external access.
- Random glass banging: calm tamas randomly charge the glass every 30-90 seconds (30% base chance). Each episode lasts ~7 seconds and deals ~20 HP total.
- Player visits reduce bang chance to 10% (within 8 units). Chance creeps back up over time.
- Bang episodes emit `tama:bang` event (for future audio) and trigger camera shake if player is within 15 units, scaled by distance.
- Tamas show agitated animation during random bangs, then return to calm wandering.

Title: Task system overhaul - interval spawning, night timer, clock, events
TL;DR: Nights now run on a real-time clock (12AM-6AM). Tasks spawn randomly from a pool on intervals instead of a fixed list. Added event scheduler for ambient scares.
---
- Replaced fixed `taskIds` with `taskPool` + `taskInterval` per night. Tasks spawn randomly every 25-75 seconds (varies by night difficulty).
- Transport is non-blocking — interval tasks start spawning immediately at night start, don't wait for transport.
- No duplicate spawns: a task from the pool can't spawn again until the previous instance is completed.
- Night ends on timer (540-600s / 9-10 min) with clean cutoff. No more "complete all tasks to end night."
- Added NightClock: maps real elapsed time to 12AM-6AM game time. HUD shows clock in top-right corner.
- Added EventScheduler: supports one-shot events at game hours and repeating events on real-time intervals.
- Each night config defines events: power surges, room flickers, random ambient flickers. Night 5 gets the most frequent and intense events.
- Events hook into existing LightingManager methods (triggerPowerSurge, triggerFlicker).

Title: Infrastructure systems - random failures, repair tasks, cascading effects
TL;DR: Four hub systems (food, water, server, generator) now randomly go offline during shifts. Each failure creates a repair task and applies gameplay effects until fixed.
---
- InfrastructureManager tracks 4 systems: food_processing, water_filtration, server_room, generator_room
- 75-second grace period at night start before systems can fail. Fail timers scale with night difficulty (harder = more frequent).
- Multiple systems can be down simultaneously
- Effects when DOWN: food processing → 2x hunger decay, water filtration → 2x thirst decay, generator → full blackout + 3x all decay (panic mode), server → disables tama care actions on device
- Bonus when UP: food/water systems give 0.5x decay rate (slower than neutral — rewards keeping systems online)
- Each failure auto-creates a repair task in the appropriate hub room using existing task types (QTE for generator, wires for server, pipes for water, holdsteady for food)
- HUD notification flash: red "[SYSTEM] OFFLINE" / green "[SYSTEM] ONLINE" messages in top-right, 3.5s duration
- Tama tab buttons show [OFFLINE] and are disabled when server room is down
- Wired into night cycle: activates at night begin with difficulty scaling, resets on death/night end

Title: Dust particles - atmospheric floating motes with flashlight interaction
TL;DR: 300 subtle dust particles drift around the player, glowing when caught in the flashlight beam. Purely cosmetic atmosphere.
---
- New `js/effects/dust-particles.js` using THREE.Points with custom ShaderMaterial
- 300 particles in a 20x20x4 box centered on the player, wrapping at edges
- Slow random drift (0.15 units/sec max) for natural floating feel
- Custom vertex shader checks if each particle falls inside the flashlight cone: particles in the beam glow brighter and grow slightly (1px → 2.5px)
- Cone detection uses dot product of flashlight direction vs particle direction with cosine angle threshold
- Proximity boost: particles very close to flashlight source (< 3 units) get extra glow regardless of cone
- Base ambient brightness of 0.04 — particles are nearly invisible in the dark, only really pop in the flashlight
- Additive blending, no depth write — particles layer on top of scene lighting naturally
- Flashlight world position and direction extracted from SpotLight + target each frame
